## 🔧 Common Issues & Fixes

**❌ dx9ware.exe is flagged or deleted by antivirus**
> This is a false positive caused by heuristic detection of injection techniques. Temporarily disable your AV, re-download from this repository, and add the executable to your AV exclusion list before re-enabling protection. Never download dx9ware from third-party sources.

**❌ Executor says "Not Attached" even after Roblox is open**
> Ensure you launched dx9ware as Administrator. Non-admin launches cannot access the Roblox process for injection. Close both applications, right-click `dx9ware.exe`, select "Run as administrator", then relaunch Roblox.

**❌ Script executes but nothing happens in-game**
> The script may be written for a different executor's API. Check if it uses non-standard function calls and refer to the dx9ware API documentation in the Help tab. Alternatively, try a script from the built-in Script Hub to confirm execution is working correctly.

**❌ GUI freezes or becomes unresponsive during injection**
> This is usually a .NET runtime version mismatch. Ensure you have .NET 6.0 or higher installed. Download the latest .NET runtime from Microsoft and reinstall, then relaunch dx9ware.

**❌ Auto-attach doesn't trigger when Roblox launches**
> Open Task Manager and confirm only one Roblox process is running. Multiple instances (from a previous crash) can confuse the auto-attach detector. End all Roblox tasks, relaunch once, and auto-attach will trigger correctly.

**❌ "Outdated version" warning appears on launch**
> dx9ware's silent updater requires an active internet connection at startup. Confirm your firewall isn't blocking dx9ware's outbound connections and restart the application to trigger the update check.

**❌ Execution history is not saving between sessions**
> The history log is stored in `%AppData%\dx9ware\`. If this folder doesn't exist or has been blocked by AV software, history cannot be written. Manually create the folder and ensure your AV is not quarantining files inside it.
